## Shell
- **SPA**: a fixed nav + left sidebar + `<main>` holding **eight** `.page` divs (Labels removed 2026-05-28). `navigate(p)` (666) toggles `.active` on `#page-<p>` and the matching sidebar `.ni[data-page]`.
- **Auth**: login screen (`#login-screen`, ~131). Token stored in `localStorage` as `wms_token`; sent as `x-wms-token` by the `api()` helper (608). A 401 from `api()` triggers `showLogin()` + throws "Session expired…".
- **Top status dot** (nav) reflects **DB health only** (honest, 2026-05-28). `setSyncing()`/`setSynced()` (626/630) show transient "Syncing…/Live". **`refreshDbStatus()` (636)** drives the authoritative state from `/api/health`'s `db` field → Live / **DB Error** (only when `db≠connected`) / Offline; it's called on init + after sign-in. **`setSyncErr(err)` (650)** classifies: a 401/"session expired" → amber **"Session expired"** (`.sync-dot.warn`); any other failure → defers to `refreshDbStatus()` (so a failed eBay sync with a healthy DB stays "Live" — surfaced via toast + per-store cards, NOT the DB dot). All 9 catch sites pass the error. Toasts via `showToast(msg,isErr)` (682).

## Key data facts (gotchas for future sessions)
- **eBay orders & listings are NOT persisted** — they live only in the in-memory JS arrays `ORDERS` and `ALL_LISTINGS`, repopulated per **live sync** and **reset on page refresh**. Inventory Health depends on `ALL_LISTINGS` being loaded first (hence the empty-state Sync button).
- **CSV import retired 2026-05-28** — the Dashboard "Quick Import" card and the Import-CSV buttons/file-inputs on eBay Orders & Listings were removed, along with `parseCSV`/`importOrdersCSV`/`importListingsCSV`. CSV import was fully client-side (FileReader) — no backend upload route existed. **Export CSV is retained** (Admin `exportCSV` + Inventory Health `exportHealthCSV`).
- **Multi-store:** every `ORDERS`/`ALL_LISTINGS` entry carries a `store` tag (`dynatrack`/`autolumen`). `syncEbayOrders`/`syncEbayListings` hit the combined endpoints (full replace — both stores). **`syncStore(key)` (per-store) MUST replace only that store's slice via `arr.filter(x => x.store !== key).concat(fresh)` — never reassign the array, or the other store is silently wiped** (commented at the call site). `storeBadge`/`storeLabel`/`storeCountLabel` render store chips/counts.
- **SKU normalization** (`normalizeSkuKey`, 1254): `(sku||'').trim().toUpperCase().replace(/[A-Z]+$/,'')` — strips trailing letters so eBay `INT4306R` matches WMS serial `INT4306` (rule 8).
- All data access goes through `api(path,opts)` (608) → `fetch('/api'+path)` with the token header. No third-party SDKs (rule 19).
- **Top status dot = DB health only** (`refreshDbStatus`, 636). It is NOT a generic op spinner: eBay/auth/network failures must not turn it red "DB Error". A 401 → amber "Session expired"; non-DB op failures defer to `/api/health.db`. Only `db≠connected` shows "DB Error". (Fixed 2026-05-28 — `setSyncErr` used to hardcode "DB Error" for any caught failure.)
- **Labels page removed 2026-05-28** (serials/barcodes are minted in a separate external system, scanned in via Scan). With it gone, two backend routes have **no frontend caller**: `POST /api/sequences/next/:prefix` and `GET`+`POST /api/print-log` (orphaned — left in server.js pending a separate decision). `GET/POST/PATCH /api/sequences` are still used by Admin's Serial Sequences section. The `.lp*` label-preview CSS classes are now dead (harmless, left in place).
